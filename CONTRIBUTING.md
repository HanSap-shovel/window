# 함께 깎는 법 — 기여 가이드

이 저장소는 답을 주려 쓰는 것이 아니라, **답을 찾는 구조**를 함께 깎기 위해 쓰입니다.
당신에게 맞는 자리가 아래 네 가지 중 하나 이상 있을 것입니다.

---

## 네 가지 참여 방식

1. **읽는다** — 그저 읽어 주십시오. 이것만으로도 충분합니다.
2. **질문·모순을 지적한다** (Issue) — 체계 안의 빈 자리, 흐린 어휘, 앞뒤 맞지 않음을 짚어 주십시오.
3. **번역한다** — 이 체계가 더 많은 언어로 열리려면 당신의 손이 필요합니다.
4. **자기만의 2단 무기를 짓고 공유한다** — 이 구조 위에 각자가 자기 도구를 짓습니다.

---

## 다섯 가지 규칙

이 규칙을 벗어나는 기여는 받지 않습니다.

1. **방향은 저자가 정합니다.** 당신의 2단은 당신 것이지만, 1단(책·체계)의 방향은 저자가 지킵니다.
2. **거짓말만 쓰지 않습니다.** 정직이 유일한 규칙입니다.
3. **이전 답은 지우지 않습니다.** 이전 답의 오류를 발견했다면, 덧붙이되 지우지 마십시오. 변화의 궤적이 이 프로젝트의 본체입니다.
4. **추상어로 도망치지 않습니다.** 구체 사례·장면·경험으로 내려옵니다.
5. **한 번에 끝내지 않습니다.** 작은 삽으로, 여러 번.

---

## Issue 열기

세 가지 라벨이 있습니다. 라벨에 맞는 템플릿을 선택해 주십시오.

- **`tension`** — 체계 안에서 모순·빈 자리·흐린 어휘를 발견했을 때.
- **`weapon`** — 이 구조 위에 자기만의 도구를 지었을 때.
- **`translation`** — 번역을 시작하고 싶을 때 (중복 방지).

---

## 번역 가이드

### 파일 규칙
- README 3 언어는 `README.md` (한국어 원본) · `README.fr.md` · `README.en.md`.
- 다른 언어는 `README.<lang>.md` 규칙 (예: `README.es.md`, `README.ja.md`).
- 문서 번역: `docs/persona/` 안에 원본 파일명 + 언어 접미사 (예: `00-synthesis-es.md`).

### 용어 사전 — 건드리지 말 원어들

아래 어휘는 **원어를 유지**해 주십시오. 괄호 안에 의미 풀이를 병기할 수는 있지만, 다른 단어로 대체하지 마십시오.

| 원어 | 의미 | 처리 예 |
|---|---|---|
| 창(窓) · *chang* | 각자를 깊이 들여다보는 도구 — 체계의 중심 은유 | *the window (窓, chang)* / *la fenêtre (窓, chang)* |
| 한 삽 · *HanSap* | 저자의 기질·가명 — 매일 한 번의 작은 삽질 | *one shovel (한 삽)* / *une pelletée (한 삽)* |
| 만족(滿足) · *manjok* | 가득 차고 발이 멈추는 상태 — 욕망의 유일한 끝 | 원어 + 의미 병기 |
| 살신성인(殺身成仁) · *sal-shin-seong-in* | 자기 몸을 희생해 사랑을 이룸 | 원어 + 의미 병기 |
| 2단 무기 · *second stage* | 독자가 1단 책 위에 짓는 자기만의 도구 | 첫 등장 시 원어 병기 |
| 뛰어 · *run!* | 2002 년 10 월 탈북의 밤, 새아버지의 외침 | 장면 속 대사는 문맥에 맞게 번역 |

### 레지스터
- 프랑스어: **vous** (공식, 따뜻한 거리)
- 영어: **you** (중립)
- 톤: 선배·사촌이 건네는 이야기. **저자·설교자·전문가 톤 금지.**

## Pull Request 규칙

- **작은 커밋** — "한 삽" 원리. 한 PR = 한 가지 변경.
- **커밋 메시지는 한국어/영어 모두 가능.** 단 이유가 보이게.
- **README 또는 핵심 철학 문서를 직접 수정하지 마십시오.** 먼저 Issue 로 토론.
- **오타·문법·링크 깨짐은 직접 PR** 환영.

---

## 2단 무기 공유에 대해

당신이 이 구조 위에 지은 자기 도구를 공유하는 자리는 **Discussions** (향후 활성화 예정) 입니다.
지금은 `weapon` 라벨로 Issue 를 먼저 열어 주십시오. Discussions 가 열리면 그쪽으로 이전됩니다.

---

## 저자의 약속

당신이 보낸 Issue · PR 에 며칠 또는 몇 주가 걸릴 수 있습니다. 저자는 세 아이의 아버지이자 다른 일을 하는 사람이고, 이 저장소는 *"한 삽"* 의 속도로 움직입니다.

그러나 모든 기여는 읽힙니다. 거절될 경우에도 이유가 있습니다.
수락 여부보다 **당신이 이 자리에 온 사실** 이 이 체계의 작동 증거입니다.
