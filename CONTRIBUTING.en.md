# How to Shape This Together — Contribution Guide

This repository is not written to hand out answers. It is written so that a **structure for finding answers** can be shaped together.
Among the four ways below, at least one seat is likely yours.

---

## Four ways to take part

1. **Read.** Simply reading is enough.
2. **Point out a question or a contradiction** (via Issues) — an empty seat, a vague term, something that does not add up.
3. **Translate.** The system opens to more languages only through your hands.
4. **Build your own second-stage tool and share it** — each reader builds their own weapon on top of this structure.

---

## Five rules

Contributions outside these five will not be accepted.

1. **Direction is set by the author.** Your second stage is yours, but the direction of the first stage (the book, the system) is kept by the author.
2. **Do not lie.** Honesty is the only rule.
3. **Do not erase earlier answers.** If you find an error in an earlier answer, add to it — do not remove it. The trajectory of change is the body of this project.
4. **Do not flee into abstraction.** Come down to concrete scenes, cases, experiences.
5. **Do not finish in one go.** Small shovelfuls, many times.

---

## Opening an Issue

Three labels. Pick the template matching the label.

- **`tension`** — you found a contradiction, an empty seat, a vague term.
- **`weapon`** — you built your own tool on top of this structure.
- **`translation`** — you want to start translating (declare first to avoid duplication).

## Translation guide

### File rules
- The three README files are `README.md` (Korean, original), `README.fr.md`, `README.en.md`.
- For another language, follow `README.<lang>.md` (e.g., `README.es.md`, `README.ja.md`).
- For other documents, place them under `docs/persona/` with a language suffix on the filename (e.g., `00-synthesis-es.md`).

### Terminology — words to leave in the original

Keep the following words in their original form. You may add a gloss in parentheses, but do not replace them with a different word.

| Original | Meaning | Example handling |
|---|---|---|
| 창(窓) · *chang* | The tool that lets each person look inward deeply — the central metaphor | *the window (窓, chang)* / *la fenêtre (窓, chang)* |
| 한 삽 · *HanSap* | The author's temperament and pen name — one shovelful each day | *one shovel (한 삽)* / *une pelletée (한 삽)* |
| 만족(滿足) · *manjok* | The state of being full, with feet at rest — the only end of desire | Original + gloss |
| 살신성인(殺身成仁) · *sal-shin-seong-in* | To offer one's body in order to realize love | Original + gloss |
| 2단 무기 · *second stage* | The tool each reader builds on top of the first-stage book | Original on first use |
| 뛰어 · *"Run!"* | The cry of the stepfather on the night of the 2002 October crossing | Translate as dialogue fits the scene |

### Register
- French: **vous** (formal, warm distance)
- English: **you** (neutral)
- Tone: an older sibling or cousin speaking. **Avoid authorial / preacher / expert tone.**

## Pull Request rules

- **Small commits** — the "one shovel" principle. One PR = one change.
- **Commit messages in Korean or English.** Just make the *why* visible.
- **Do not edit the README or the core philosophical documents directly.** Open an Issue first.
- **Typos, grammar, broken links** — direct PRs welcome.

## The author's promise

Your Issue or PR may wait days or weeks. The author is a father of three with other work, and this repository moves at the speed of *"one shovel."*

But every contribution is read. When a contribution is declined, the reason is stated.
More important than acceptance — **the fact that you showed up at this seat** is proof that the system is working.
